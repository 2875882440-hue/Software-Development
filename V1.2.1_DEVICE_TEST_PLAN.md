# V1.2.1 多机型测试计划

## 目的

使用同一流程比较不同品牌和 Android 版本的通知监听稳定性。测试记录不得包含完整通知正文、商户、金额、账单明细或备注；出现问题时优先分享 APP 内生成的脱敏问题日志。

## 标准步骤

| # | 测试项 | 操作 | 预期 | 结果 |
|---:|---|---|---|---|
| 1 | 覆盖安装 | 使用 `adb install -r` 或直接覆盖安装 | 原有账单与设置保留 |  |
| 2 | 启动 | 打开 APP，查看四个底部 Tab | APP 正常启动，无崩溃和乱码 |  |
| 3 | 通知监听 | 打开系统通知监听并启用芽芽记账 | 监听页显示已授权 |  |
| 4 | 自动监听 | 开启自动监听 | 前台通知保持运行 |  |
| 5 | 本 App 测试通知 | 进入测试通知流程 | APP 能捕获 probe |  |
| 6 | 微信普通通知 | 让另一台设备发送微信消息 | 通知日志出现微信记录 |  |
| 7 | 微信付款码 | 完成一次付款码支付 | 自动生成待确认账单或有明确日志 |  |
| 8 | 微信扫码 | 扫描商家码支付 | 能自动记录；无通知时使用扫码补录 |  |
| 9 | 支付宝支付 | 完成一次支付宝支付 | 自动生成待确认账单或有明确日志 |  |
| 10 | 成功率报告 | 打开监听页“成功率” | 计数按来源增加，不显示隐私内容 |  |
| 11 | 锁屏一小时 | 锁屏一小时后再支付 | 监听仍能捕获 |  |
| 12 | 手机重启 | 重启后打开 APP 并支付 | 自动监听可恢复 |  |
| 13 | 省电模式 | 开启省电模式后重复支付 | 记录是否失效及修复结果 |  |
| 14 | 权限恢复 | 关闭再开启通知监听 | 一键修复 / 重新授权可恢复 |  |
| 15 | 次日检查 | 第二天不重装直接支付 | 监听仍正常 |  |
| 16 | 问题日志 | 复制脱敏问题日志 | 包含匿名统计，不含账单隐私 |  |

## 通过标准

- 至少 4 个品牌完成标准步骤。
- 覆盖安装无数据丢失。
- 微信、支付宝普通付款通知可稳定进入 APP。
- 锁屏、重启和次日测试有明确结果。
- 失败时能够通过一键修复或重新授权恢复，或形成可复现的已知问题。
