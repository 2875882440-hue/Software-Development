# CHANGELOG

## V1.2.0 · UI Refresh 回归封存

版本号：`1.2.0`
版本码：`120`

### 回归与封存

- 完成记账、统计、限额、监听四个底部 Tab 及主要二级页面的真机回归。
- 恢复 UI 替换后遗漏的来源统计、非空周/月归档和限额剩余/超额金额展示。
- 验证手动记账、待确认、账单详情、账单编辑、删除入口和备份恢复入口完整保留。
- 验证 CSV 导出、完整 JSON 备份、JSON 恢复及重复账单跳过逻辑入口和单元测试保持正常。
- 验证自动监听开关、通知监听权限、前台服务、健康诊断、监听急救、监听应用、问题日志和后台报告入口完整保留。
- 系统返回和顶部返回均能从子页面回到父级页面，不会直接退出 App。
- Room schema 保持版本 12，未修改数据库、通知监听、金额解析或备份恢复核心代码。
- 84 项单元测试、Debug APK 构建和 Xiaomi 2510DRK44C 覆盖安装通过。

### 品牌与界面

- 按 `F:\Software Development\1` 的 14 张导出稿补齐首次使用、统计、每日限额、自动监听和系统设置页面。
- 首次启动新增三步本地自动记账引导，完成后直接进入系统通知监听授权流程。
- 统计页新增真实账单驱动的收支概览、趋势柱状图和分类环图，不使用设计稿示例数据。
- 每日限额页新增环形进度、真实分类支出、当日静音开关和统一限额表单。
- 自动监听页重排为运行结论、权限设置、运行状态和诊断补救，原测试、日志与修复入口全部保留。
- 系统设置整合监听应用开关、后台权限状态和系统设置入口。
- 再次以 `design-prototypes` 的“青芽日常”原型代码为唯一视觉基准校准 UI：迁移原型吉祥物矢量资源，统一标题字体层级，补齐截图重选、规则折叠编辑、备份卡片和关于信息。
- 全部工具收敛为原型定义的 8 个主要入口，额外诊断能力继续保留在监听与诊断页面。
- 软件正式名称由“本地自动记账”更新为“芽芽记账”。
- 采用用户选定的“青芽日常”视觉方向。
- 更新启动图标为绿色账本与嫩芽形象。
- 新增“芽芽”界面角色，用于首页与监听状态沟通。
- 全页面统一奶油白背景、森林绿主色、大圆角卡片与青芽状态色。
- 首页改为月度结余、消费节奏、快捷记账和最近账单的信息层级。
- 根据 `design-prototypes/index.html` 重新校准首页结构：日期问候、紧凑监听条、绿色金额 Hero、四色快捷四宫格和三条最近账单。
- 新增与原型一致的独立“全部工具”两列卡片页，并在工具页隐藏底部导航。
- 新增/编辑账单改为分段收支选择、大金额输入、分类胶囊与紧凑表单卡。
- 底部导航增加清晰图形标识，并把“记一笔”放在中央主操作位置。
- 监听页默认展示易懂状态，技术诊断字段收进“查看详细状态”。
- 待确认、账单详情、截图识别、备份恢复、分类规则、学习记录、监听工具与诊断页统一为青芽日常组件。
- 二级页面统一增加状态栏安全区，避免标题与系统状态栏重叠。
- 浅色页面启用深色系统状态栏与导航栏图标，保证时间、信号和系统导航清晰可见。
- 统计范围按钮改为等宽单行布局，避免“自定义”在窄屏真机上竖排。
- 已在 Xiaomi 2510DRK44C 真机覆盖安装并完成主流程验证。
- 修复首页“今日支出”误用累计支出的问题，现按自然日范围实时统计。
- 首次使用引导期间不再弹出自动监听启动/恢复提示，避免遮挡引导流程。
- 真机验证自动监听恢复后前台服务保持运行，原有账单数据库与待确认数据完整保留。
- 未修改 Room 数据库、通知解析、限额计算或备份恢复核心逻辑。
