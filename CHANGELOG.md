# CHANGELOG

## V1.0.0 Personal Stable

版本号：`1.0.0`  
版本码：`100`

### 封存内容

- 封存为个人稳定版。
- 保留本地记账、通知监听、统计、限额、备份恢复等完整个人使用流程。
- 不新增功能，不修改数据库结构，不调整核心监听、解析、限额、备份恢复逻辑。
- 页面版本显示更新为 `V1.0.0`。

### 当前稳定能力

- 本地手动记账。
- 微信/支付宝付款通知监听。
- 自动生成账单和待确认账单。
- 无金额通知待补录。
- 账单编辑、删除与重复过滤。
- 自动分类和自定义分类规则。
- 日 / 周 / 月统计。
- 每日限额提醒和今日不再提醒。
- 自动监听状态检测。
- 前台服务保活。
- 监听急救 / probe 检测。
- 手势返回。
- CSV 导出。
- JSON 完整备份。
- JSON 恢复并跳过重复账单。

## V7.0.1 Hotfix

- 修复 V7.0 真机测试后出现的大量中文 UI 乱码。
- 修复金额格式化模板损坏导致的异常显示，例如乱码金额和 `,.2f` 类错误。
- 统一金额显示为 `¥xx.xx` / `-¥xx.xx`。
- 修复底部 Tab 文案为“记账 / 限额 / 监听”。
- 修复备份恢复、限额、监听、统计、记账表单等主要页面文案。

## V7.0

- 整合主界面为“记账 / 限额 / 监听”三 Tab。
- 增加备份与恢复入口。
- 增强统计卡片、限额页和监听页的信息展示。
- 真机测试核心流程通过，但 UI 文案在该版本出现编码损坏，后续由 V7.0.1 修复。

## V6.x

- 增强自动监听状态检测。
- 增加前台服务保活。
- 增加监听健康检查、后台诊断报告和一键修复。
- 增加 requestRebind 自恢复尝试。
- 增强真机测试流程，重点验证微信/支付宝通知监听。
- 增加日 / 周 / 月统计、分类汇总、来源汇总和按时间分组账单。

## V5.x

- 增加限额提醒。
- 支持每日限额、今日已消费、剩余额度和超额提醒。
- 支持“今日不再提醒”。
- 完善账单编辑、删除和待确认流程。

## V4.x

- 增加截图记账和补录相关流程。
- 支持无金额通知进入待补录。
- 增强自动分类和分类规则。

## V3.x

- 完成微信/支付宝付款通知识别。
- 支持从真实通知中解析金额、商户、来源和收支类型。
- 增加待确认账单，降低误记风险。

## V2.x

- 引入通知监听能力。
- 支持自动监听付款通知并生成候选账单。
- 开始进行真机通知监听测试。
