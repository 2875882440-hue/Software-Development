# 芽芽记账 V1.2.1 发布说明

## 版本定位

V1.2.1 是基于 V1.2.0 UI Refresh 的稳定化维护版，不增加账号、云同步或大型页面，重点提升项目一致性、可量化测试和问题诊断能力。

## 本次更新

- 新增最近 30 天本地匿名自动记账计数。
- 新增最近 14 天总体、微信、支付宝、其他应用成功率报告。
- 统计通知接收、支付识别、自动生成、金额失败、重复过滤、确认、改金额和删除。
- 问题日志增加匿名统计汇总，继续保护通知和账单隐私。
- 统一项目版本文档和维护边界。
- 新增多机型统一测试计划与反馈模板。
- 生成固定版本 Debug APK 和 SHA-256 校验值。

## 隐私说明

成功率统计只保存自然日、来源分组、事件类型和整数计数。不会保存：

- 通知标题或正文
- 商户名称
- 金额
- 备注
- 账单 ID 或账单内容
- 用户账号或设备身份

## 已知限制

- 微信、支付宝或其他支付 APP 没有发送系统通知时，芽芽记账无法自动记录。
- 通知内容被隐藏、折叠或脱敏时，金额和商户解析可能失败。
- 不同厂商的省电和后台策略可能中断通知监听，需要手动允许自启动、后台活动和电池无限制。
- 本版本封存的是 Debug APK，适合个人和小范围测试，不等同于应用商店正式签名版本。

## 升级与回退

- 建议升级前先在 APP 内导出 JSON 完整备份。
- V1.2.1 未修改 Room schema，覆盖安装不应清空账单。
- 不建议卸载后回退；卸载会删除本机应用数据。
- 如 V1.2.1 出现问题，保留现有数据并覆盖安装已封存的 V1.2.0 APK。

## 测试重点

请按照 `V1.2.1_DEVICE_TEST_PLAN.md` 验证覆盖安装、微信付款码、微信扫码、支付宝、锁屏、重启、次日监听和问题日志。
