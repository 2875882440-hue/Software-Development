## 本版本目标

新增“导出问题日志”功能，便于用户在遇到监听异常、无法自动记账、付款未记录等问题时，将安全的诊断文本复制或分享给开发者排查。

本版本不新增云同步、登录、服务器、无障碍服务、截屏监听；不修改数据库结构；不清空账单；不改动支付解析核心逻辑。

## 修改摘要

- 在【监听】页新增“导出问题日志”卡片。
- 在【后台诊断报告】页新增同样的“导出问题日志”入口。
- 新增“复制问题日志”功能：
  - 生成纯文本问题日志。
  - 写入 Android 剪贴板。
  - 提示“问题日志已复制，可以发送给开发者。”
- 新增“分享问题日志”功能：
  - 调用 Android 系统分享面板。
  - 以纯文本方式分享诊断内容。
- 新增 `ProblemLogExporter`，集中生成脱敏问题日志。
- 新增 `ProblemLogExporterTest`，验证日志包含诊断字段且不包含隐私数据。
- 更新页面显示为 `V1.1.3`。
- 增加 `BuildConfig.BUILD_TIME`，导出日志中显示构建时间。

## 问题日志字段

导出文本包含：

- APP 信息：
  - APP 版本号
  - versionCode
  - 构建时间
  - 导出时间
- 设备信息：
  - 品牌
  - 厂商
  - 型号
  - 设备代号
  - Android / HarmonyOS / 系统版本
  - SDK 版本
  - ROM 类型
  - 是否华为 / 荣耀设备
- 监听状态：
  - 通知监听权限是否开启
  - 自动监听是否开启
  - 前台服务是否运行
  - NotificationListenerService 是否 connected
  - raw listener connected 状态
  - 当前健康状态
  - 健康原因
  - 最近健康检查时间
  - 最近 requestRebind 结果
  - 最近失败原因
- 最近记录：
  - 最近通知时间
  - 最近微信通知时间
  - 最近支付宝通知时间
  - 最近付款通知时间
  - 最近 probe 成功时间
  - 最近 probe 失败时间
- 后台诊断统计：
  - 最近 24 小时健康检查次数
  - probe 成功次数
  - probe 失败次数
  - requestRebind 次数
  - 监听断开次数
  - 前台服务启动次数
  - 前台服务停止次数
- 诊断结论：
  - 监听正常
  - 通知监听权限未开启
  - 自动监听未开启
  - 前台服务未运行
  - 权限已授权但监听服务未连接
  - 疑似厂商系统限制后台监听
  - 最近没有收到微信 / 支付宝通知
- 建议操作：
  - 重新授权通知监听
  - 开启自动监听
  - 恢复前台服务
  - 检查后台活动、电池优化、通知权限
  - 检查微信 / 支付宝通知权限

## 隐私说明

默认导出的问题日志不包含：

- 完整账单数据
- 完整通知正文
- 商户名称
- 金额明细
- 备注
- 用户个人隐私信息

单元测试已覆盖：即使运行时状态中存在通知标题、通知文本、商户名、金额、备注，导出文本也不会包含这些隐私内容。
