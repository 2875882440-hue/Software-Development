# BUG_LOG

## V1.0.0 Personal Stable 已知问题

- 不同 Android / MIUI / HyperOS 机型仍可能限制后台监听。
- 微信/支付宝如果不发付款通知，APP 无法自动记录。
- 微信/支付宝通知内容被隐藏、折叠或脱敏时，可能影响解析。
- 用户仍需手动开启通知监听权限和自动监听。
- 系统重启、系统省电策略或厂商后台管理可能导致前台服务被停止。
- 部分极端场景可能需要手动补录。
- 支付平台通知文案变化时，解析准确率可能下降。

## V7.0.1 已修复问题

- 修复 V7.0 大量中文 UI 文案乱码。
- 修复金额格式化模板损坏导致的异常金额显示。
- 修复底部 Tab 显示“标签”的问题。
- 修复备份恢复、限额、监听、统计、记账表单等主要页面文案异常。

## 处理原则

- 优先修复影响个人稳定使用的小 Bug。
- 不因小问题重写项目。
- 不修改 Room 数据库结构。
- 不清空账单数据。
- 不随意调整通知监听、支付解析、限额、备份恢复核心逻辑。

## V1.1.0 已修复问题

- 修复部分付款通知金额被截成第一位的问题，例如 37 元误识别为 3 元。
- 增加金额候选诊断字段：`amountCandidates`、`selectedAmount`、`selectedReason`、`failReason`。
- 增加商户分类学习记录，用于根据历史手动确认结果推荐分类。
- 统计功能提升为底部独立 Tab。
