# BUG_LOG

## V1.2.1 当前已知问题

- 不同 Android / MIUI / HyperOS 机型仍可能限制后台监听。
- HarmonyOS、ColorOS、OriginOS、One UI 等系统尚缺统一长期测试。
- 微信/支付宝如果不发付款通知，APP 无法自动记录。
- 微信/支付宝通知内容被隐藏、折叠或脱敏时，可能影响解析。
- 用户仍需手动开启通知监听权限和自动监听。
- 系统重启、系统省电策略或厂商后台管理可能导致前台服务被停止。
- 部分极端场景可能需要手动补录。
- 支付平台通知文案变化时，解析准确率可能下降。
- 最近 14 天成功率只统计 APP 实际收到的通知，无法计算支付软件未发送通知造成的漏记。
- 当前封存产物为 Debug APK，正式签名 Release APK / AAB 流程尚未完成。

## V1.2.1 已完成改进

- 增加最近 30 天每日匿名自动记账计数，不保存通知正文、商户、金额、备注或账单内容。
- 增加最近 14 天总体、微信、支付宝、其他应用成功率报告。
- 记录通知漏斗、金额解析失败、重复过滤、用户确认、用户修改金额和用户删除。
- 将匿名统计附加到脱敏问题日志，便于远程判断损失环节。
- 统一版本文档和多机型测试模板。

## V7.0.1 已修复问题

- 修复 V7.0 大量中文 UI 文案乱码。
- 修复金额格式化模板损坏导致的异常金额显示。
- 修复底部 Tab 显示“标签”的问题。
- 修复备份恢复、限额、监听、统计、记账表单等主要页面文案异常。

## 处理原则

- 优先修复影响个人稳定使用的小 Bug。
- 不因小问题重写项目。
- 不修改 Room 数据库结构。
- 不清空账单数据。
- 不随意调整通知监听、支付解析、限额、备份恢复核心逻辑。
- 对成功率下降必须先结合匿名统计、通知日志和真机步骤定位，避免盲目修改解析器。
