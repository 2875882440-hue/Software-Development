## 当前已知限制

- 支付软件没有发送可读取系统通知时，APP 无法自动记录，也无法统计未到达 APP 的付款。
- 通知内容隐藏、折叠、脱敏或文案变化时，金额和商户解析可能失败。
- MIUI、HyperOS、HarmonyOS、ColorOS、OriginOS 等系统可能限制后台监听。
- 用户仍需手动授权通知监听、通知权限、自启动、后台活动和电池无限制。
- 最近 14 天成功率反映“APP 实际收到的通知”处理效果，不等于全部真实付款覆盖率。
- 当前主要验证 Debug APK，尚未完成应用商店正式签名和 AAB 发布流程。

## 维护边界

- 不大规模重写 UI。
- 不随意升级 Room schema 或清空账单。
- 不重写通知监听、支付解析、限额和备份恢复核心逻辑。
- 不加入云同步、登录、会员、服务器或持续截屏 / 无障碍监听。
- 新功能必须优先服务稳定性、可诊断性和个人长期使用。
