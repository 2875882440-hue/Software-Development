# V1.2.1 稳定化封存设计

## 目标与边界

V1.2.1 不扩展云端、账号、会员或大型 UI 能力，只完成四项稳定化工作：统一项目文档；增加本地匿名自动记账统计；建立多机型测试流程；生成可验证的固定版本 APK 与封存资料。

本版本继续保持 Room schema 12，不迁移账单数据库，不修改通知金额解析算法、重复判断算法、限额算法和备份格式。自动记账统计只保存计数，不保存通知标题、正文、商户、金额、备注、账单 ID 或用户身份信息。

## 匿名统计设计

统计数据使用独立 `SharedPreferences` 保存最近 30 个自然日的每日计数。每个计数键只包含日期、来源分组和事件类型。来源分为微信、支付宝、其他已启用监听应用；事件分为收到监听通知、识别为支付相关、生成待确认账单、金额解析失败、重复通知过滤、用户确认、用户修改金额和用户删除。

通知服务在确认应用已被用户勾选后记录“收到监听通知”，完成解析后记录支付相关、金额失败、生成账单或重复过滤。ViewModel 在用户对通知来源账单执行确认、编辑金额或删除后记录对应事件。模拟通知、截图 OCR、手动记账和本 App probe 不计入真实自动记账统计。

默认报告窗口为最近 14 天，底层保留 30 天以便观察变化。生成成功率定义为“自动生成待确认账单数 / 支付相关通知数”；通知转化率定义为“自动生成待确认账单数 / 收到的已监听应用通知数”。分母为零时显示暂无数据，不显示误导性的 0%。

## 展示、诊断与隐私

监听页诊断工具区新增“成功率”入口。报告页显示最近 14 天总体漏斗、微信/支付宝/其他应用分项结果以及统计口径说明。问题日志追加同一份匿名汇总，只输出数字和来源分组，不输出具体应用通知内容或账单信息。

统计存储通过小型存储接口与计算器分层：Android 层负责 SharedPreferences，纯 Kotlin 层负责键解析、30 天清理、14 天聚合和成功率计算。单元测试覆盖跨日聚合、来源拆分、过期清理、零分母和隐私字段约束。

## 测试与封存

自动验证依次运行 `testDebugUnitTest`、`compileDebugKotlin` 和 `assembleDebug`。构建产物复制为固定名称的 V1.2.1 Debug APK，并生成 SHA-256。多机型测试文档统一记录品牌型号、系统版本、覆盖安装、权限、微信付款码、微信扫码、支付宝、锁屏、重启、次日监听、一键修复与问题日志结果。

没有连接真机时，构建与文档工作继续完成，真机支付链路在测试报告中明确标为待人工验证。GitHub Release 属于外部发布动作，只在本地封存通过、仓库认证可用且发布内容确认后执行。
