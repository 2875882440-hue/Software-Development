# 自动记账 APP 全量 UI 设计

## 设计目标

在不改动记账、通知监听、支付解析、限额、数据库和备份恢复逻辑的前提下，重做整个 APP 的视觉与信息架构。新界面以现有《自动记账 APP 使用说明》为品牌参考：绿色、圆润、轻松、插画化，同时保证账单数据清晰、监听状态易懂、异常修复路径明确。

第一阶段交付独立交互原型，不直接修改 Android 业务源码。原型覆盖全 APP 页面，并提供三套可以即时切换的视觉方向。用户选定方向后，再迁移到 Jetpack Compose。

## 品牌与视觉方向

### 方案 A：青芽日常（推荐）

- 奶油白背景、森林绿主色、嫩绿色辅助色、黄色提醒色。
- 白色圆角卡片带轻柔绿色阴影，接近使用说明的亲切插画风。
- 首页强调“自动记账是否正常”“今天花了多少”“还有多少预算”。
- 适合长期日常使用，品牌一致性最好。

### 方案 B：阳光票据

- 暖米色背景、深苔绿主色、阳光黄与珊瑚橙作为重点色。
- 卡片加入票据虚线、日期章和轻微纸张层次，强调生活记录感。
- 新增账单、待确认和账单详情更像整理票据。
- 视觉性格更强，但数据页需要克制装饰密度。

### 方案 C：薄荷数据

- 冷白背景、深墨绿主色、薄荷绿与湖蓝辅助色。
- 图表、状态标签和数据分组更明确，卡片更加紧凑。
- 统计、限额、监听与诊断页的信息效率最高。
- 更专业，但情绪温度略低于前两套。

## 吉祥物

设计一个由“小账本 + 绿芽”组成的圆润小角色，暂名“芽芽”。它不是装饰性贴纸，而是状态沟通组件：

- 首次使用：带用户完成监听授权。
- 空账单：提示“第一笔账，从今天开始”。
- 监听正常：开心状态。
- 权限缺失或监听异常：举起黄色提醒牌。
- 备份成功、账单保存成功：绿色勾选状态。
- 错误与故障诊断：保持友好，不使用恐吓式红色大警告。

原型使用可缩放 SVG 绘制，Android 落地时可转换为 VectorDrawable 或保留为受控矢量资源。

## 信息架构

底部导航保持四个主要入口：

1. 记账：首页、最近账单、待确认、快速记账。
2. 统计：今日、本周、本月、自定义范围及分类/来源分析。
3. 限额：今日预算进度、每日限额设置、提醒状态。
4. 监听：自动监听总开关、通知权限、运行状态与一键修复。

中间悬浮“记一笔”按钮作为全局主操作。低频能力统一放入“全部工具”：截图记账、扫码支付后补录、分类规则、学习记录、备份与恢复、监听诊断、问题日志、监控应用和后台设置。

## 页面覆盖

- 首次使用与权限引导
- 记账首页
- 全部账单与分组筛选
- 手动新增、编辑与账单详情
- 待确认账单
- 截图识别预览
- 快速补录与扫码支付后补录
- 统计首页、分类统计、来源统计与周期归档
- 每日限额与超额状态
- 自动监听、权限状态、一键修复与健康报告
- 真实通知测试、微信扫码测试和原始通知详情
- 分类规则与学习记录
- 监控应用与后台运行设置
- 备份、恢复与问题日志导出
- 空白、加载、成功、错误、权限缺失和服务中断状态

## 交互原则

- 每个页面只保留一个最明显的主操作。
- 技术状态翻译成用户语言，例如“监听服务未运行”改为“自动记账暂时休息了”。
- 危险操作使用二次确认；删除与覆盖恢复不靠颜色作为唯一提示。
- 触控目标不小于 44dp，正文对比度满足 WCAG AA。
- 所有图标配文字标签；底部导航、表单和对话框支持键盘焦点状态。
- 空态说明下一步，错误态同时提供恢复动作。

## 原型架构

原型放在 `design-prototypes/`，使用无外部依赖的 HTML、CSS 和 JavaScript。三套视觉共用一套语义化页面结构和交互模型，通过设计令牌切换主题，避免重复页面和内容偏差。原型用于视觉评审，不连接真实数据库，也不会读取或修改用户账单。

## 验收标准

- 三套主题可在同一入口即时切换。
- 至少 13 个代表页面可导航，覆盖现有 APP 的全部功能类型。
- 390px 手机宽度无横向滚动，桌面上以手机画板展示。
- 中文无乱码，金额采用 `¥1,280.00` 格式。
- 主流程可点击：首次引导 → 监听 → 首页 → 新增账单 → 账单详情。
- 可预览正常、待确认、空白、错误和权限缺失状态。
