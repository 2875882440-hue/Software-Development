# 青芽日常原型忠实迁移设计

## 目标

以 `design-prototypes/index.html` 的第一个“青芽日常”方案为唯一视觉基准，将页面结构和组件密度迁移到正式 Jetpack Compose 应用，而不只复用绿色配色。所有金额、账单、监听状态和工具入口继续使用现有 ViewModel 与业务回调。

## 页面结构

首页按照原型固定为：日期问候、自动监听状态条、绿色月度金额 Hero、四个快捷记账入口、三条最近账单和悬浮圆角底栏。“全部工具”使用独立的两列工具卡页面，并在该页面隐藏底栏。新增/编辑账单采用支出收入分段、大金额输入、分类胶囊和紧凑表单卡。

统计、限额和监听沿用真实状态与错误处理，但统一使用原型的 16dp 页面边距、23sp 标题、10sp 辅助文案、24dp 卡片圆角、绿色主按钮和原型式底部导航。二级页面统一为左侧标题、右侧方形返回按钮。

## 数据与错误处理

UI 不创建演示数据。首页金额来自现有月度统计，最近账单来自已确认账单，待确认数量来自真实状态。监听异常继续显示真实修复入口；截图识别、备份恢复、分类规则和诊断流程保持原回调。表单校验、Room 保存、通知解析和监听恢复逻辑不变。

## 验证

运行 `testDebugUnitTest` 和 `assembleDebug`，然后使用 `adb install -r` 覆盖安装并保留数据。真机截图对照原型检查首页、全部工具、记账表单、统计、限额和监听页面，同时检查系统栏、滚动、返回与底部导航。
